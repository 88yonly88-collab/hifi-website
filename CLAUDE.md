このファイルは、Claude Codeがこのプロジェクトで作業するときに最初に読むエントリポイントです。  
ルールの正本は `AGENTS.md` です。本ファイルと `AGENTS.md` の内容が矛盾する場合は、`AGENTS.md` を正とします。  
新しいリポジトリを作るときは、このファイルをプロジェクト直下に `CLAUDE.md` としてコピーし、`[ ]` の項目を埋めてください。

## 1. 最初に読むファイル

作業開始時は、まず以下を確認してください。

1. `AGENTS.md`(ルールの正本。検証・秘密情報・報告・プロジェクト固有情報はすべてここにある)
2. `CLAUDE.md`(本ファイル。読み順とClaude Code固有の注意のみ)
3. `README.md` があれば読む
4. `CLAUDE_CODE_HANDOFF.md` があれば読む(前セッションからの引き継ぎメモ。存在する場合のみ読む。作成するのは、ユーザーから引き継ぎ指示があった時のみ)
5. `package.json` があれば scripts を確認する
6. `.env.example` があれば必要な環境変数を確認する
7. `git status` で現在の変更状態を確認する

## 2. Claude Codeの役割

Claude Codeは、このプロジェクトの実装担当です。

重視すること:

- まず関連ファイルを読む
- いきなり大規模修正しない
- 1回の修正範囲を小さくする
- 保存処理・認証・API・データ構造に触る変更は、変更前にdiff(変更予定の内容)を提示して承認を得る
- 変更後は検証する(AGENTS.md セクション5に従う)
- 検証できなかったことは「未確認」として報告する
- 非エンジニアのユーザーにもわかる日本語で説明する

## 3. モデル切り替えの確認ルール

Claude Codeは、自分でモデルを切り替えることはできません。切り替えられるのはユーザーだけです。そのため、以下の行動ルールに従ってください。

現行世代のモデル: Haiku 4.5(軽量)/ Sonnet 5(標準)/ Opus 4.8・Fable 5(上位)

以下の作業を、軽量モデル(Haiku 4.5 等)で実行中と思われる場合は、作業を始める前に「この作業は上位モデル推奨です。モデルを切り替えますか?」とユーザーに確認してください。ユーザーの回答があるまで、その作業には着手しないでください。

- 仕様設計、複数ファイルにまたがる設計変更
- 大規模リファクタ
- 本番影響のある変更(保存データの構造変更、認証、決済、重要API)
- 原因不明のエラーが続く場合

通常の調査、軽微な修正、ファイル整理、ドキュメント作成は、現在のモデルのまま進めてよいです。

## 4. 実装前の確認

実装前に、以下を短く整理してください。

- 今回の目的
- 触る予定のファイル
- 触らない方がよいファイル
- 検証方法
- リスクがある箇所

小さな明確なバグ修正であれば、長い計画を出さずに進めてかまいません。

## 7. Codexとの併用ルール

Codex連携は、下の記入欄にチーム名・エージェント名が実際に記入されている場合のみ実施してください。  
未記入(`[ ]` のプレースホルダーのまま)の場合は、Codex連携は行わないでください。プレースホルダーのまま送信することは禁止です。

役割分担:

- Claude Code: 実装担当
- Codex: レビュー、難所相談、別視点の確認

agmsg記入欄(プロジェクトごとに記入):

- team: [チーム名]
- Claude Code agent: [例: claude-driver]
- Codex agent: [例: codex-reviewer]

Codexへ送るときは、変更内容、確認してほしい観点、関連ファイルを短く添えてください。  
Codexから指摘が来た場合は、内容を確認し、必要なものだけを最小修正してください。
